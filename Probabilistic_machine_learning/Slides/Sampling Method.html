Why sample?

We need sampling techniques to exploit the generative aspect of generative models (e.g. mixture models)

In a Bayesian setting, we are even more reliant on being able to sample - because we are concerned with distributions over parameters of our model.

More generally, the need to calculate expectations arises frequently, and we need efficient tools to deal with this for arbitrary densities. We will discuss sampling techniques this week, and return to this next week when we discuss variational methods.

Basic sampling techniques

Sampling from standard distributions

Point of departure:

Assume we have a method to generate uniformly distributed (pseudo) random numbers in the range (0,1)(0,1)(0,1).

So far in the course, we've seen a number of basic distributions: Gaussian, Exponential, Beta, ...

In order for these distributions to be useful to us, we must be able to generate samples from them.

How can we do this?

Inverse transform sampling

Idea: take a sample u∼U(0,1)u \sim \mathcal{U}(0,1)u∼U(0,1), and transform it to a sample from the desired distribution.

How can we uniquely characterize a sample? Not by its density.

We can use the cummulative distribution function.

Inverse transform sampling

For a given u∼U(0,1)u \sim \mathcal{U}(0,1)u∼U(0,1), we need to find the sample value xxx in the target distribution such that their cummulative distribution is the same.

So, in order to sample using this technique, we must be able to evaluate the inverse cummulative distribution function for the target density.

This function is also sometimes called the quantile function.

Note that for U(0,1)\mathcal{U}(0,1)U(0,1), we have u=cdf(u)u = \mathrm{cdf}(u)u=cdf(u), so we can just use the sampled value uuu as our cummulative probability value.

Sampling from a Multivariate Normal

As we saw when we discussed PCA, a multivariate normal distribution can be rotated such that we get a diagonal covariance matrix

We can thus take independent normally distributed samples, and transform them to a multivariate Normal with a desired covariance matrix and mean vector.

  • Generate a vector ϵ\boldsymbol \epsilonϵ of independent samples from a univariate standard Normal distribution

Rejection sampling - How efficient is it?

From the denominator in the previous slide, we can read off the marginal probability of acceptance:

p(a)=∫zp(a∣z)q(z)=∫zp~(z)kq(z)q(z)=Zpkp(a) = \int_z p(a|z)q(z) = \int_z\frac{\tilde p(z)}{kq(z)}q(z) = \frac{Z_p}{k}p(a)=∫z​p(a∣z)q(z)=∫z​kq(z)p~​(z)​q(z)=kZp​​

The area under the envelope is kkk, so this is a ratio between the area under p~(z)\tilde p(z)p~​(z) and the area under kq(z)kq(z)kq(z).

Long story short: we achieve the best performance when kkk is as small as possible. I.e., we want a "tight" envelope.

Importance sampling is yet another sampling technique.

Why do we need expectations?

Intermezzo: Expectations

Reminder: an expectation is a weighted average of a function

E[f]=∫p(z)f(z)dz\mathbb{E}[f] = \int p(\boldsymbol z) f(\boldsymbol z) d\boldsymbol zE[f]=∫p(z)f(z)dz

It turns out, this is one of the most important operations in probabilistic machine learning.

Often, the goal of acquiring samples from a density is just to approximate an expectation by a finite sum:

Sampling-importance-resampling

We described importance sampling for the purpose of calculating expectations, but it is actually possible to acquire approximate samples from p(x)p(x)p(x) as well.

Strategy:
  1. Draw samples z(1)…z(L)z^{(1)}\ldots z^{(L)}z(1)…z(L) from a proposal distribution q(z)q(z)q(z)
  2. Calculate weights wlw_lwl​ just like in importance sampling
  3. Now consider your LLL samples as the outcomes of a discrete distribution, where the weights are the probabilities. You can draw approximate samples from p(x)p(x)p(x) by sampling from this distribution.

Markov Chain Monte Carlo (MCMC)

Often, it is difficult to come up with a good enough proposal distribution for importance or rejection sampling - in particular in high dimensions

Markov Chain Monte Carlo is a framework for overcoming these problems, making it possible to sample fruitfully from a very broad range of distributions.

Idea: we generate a series of samples, where each sample depends on the previous one. We will see that this makes it easier to come up with a meaningful proposal distribution.

What is a Markov Chain?

Markov chain: series of random variables z(1),…,z(L)\boldsymbol z^{(1)}, \ldots, \boldsymbol z^{(L)}z(1),…,z(L) which factorizes like:

p(z(1),…,z(L))=p(z(0))p(z(1)∣z(0)),…,p(z(L)∣z(L−1))p(\boldsymbol z^{(1)}, \ldots, \boldsymbol z^{(L)}) = p(z^{(0)})p(z^{(1)}|z^{(0)}), \ldots, p(z^{(L)}|z^{(L-1)})p(z(1),…,z(L))=p(z(0))p(z(1)∣z(0)),…,p(z(L)∣z(L−1))

We will often work with homogeneous Markov models, where p(z(l)∣z(l−1))p(z^{(l)}|z^{(l-1)})p(z(l)∣z(l−1)) is identical for all lll.

For convenience, we will develop the concepts in terms of discrete Markov Chains.

In this case, the p(z(l)∣z(l−1))p(z^{(l)}|z^{(l-1)})p(z(l)∣z(l−1)) conditional probabilities are referred to as transition probabilities.

Markov Chains: stationary distribution

The marginal distribution for any of the variables in the chain can be written as: p(z(l))=∑z(l−1)p(z(l)∣z(l−1))p(z(l−1))p(z^{(l)}) = \sum_{z^{(l-1)}}p(z^{(l)} | z^{(l-1)})p(z^{(l-1)})p(z(l))=z(l−1)∑​p(z(l)∣z(l−1))p(z(l−1))

If a marginal distribution p(z(l))p(z^{(l)})p(z(l)) is invariant wrt the transition probabilities, then it is referred to as a stationary distribution of the Markov Chain.

For a homogeneous Markov Chain, for a transition z′→zz' \rightarrow zz′→z we can write this as:

p∗(z)=∑z′p(z∣z′)p∗(z′)p^*(z) = \sum_{z'}p(z|z')p^*(z')p∗(z)=z′∑​p(z∣z′)p∗(z′)

Markov Chains: detailed balance

We have: p∗(z)p^*(z)p∗(z) is a stationary distribution of a Markov chain if:

p∗(z)=∑z′p∗(z′)p(z∣z′)p^*(z) = \sum_{z'}p^*(z')p(z|z')p∗(z)=z′∑​p∗(z′)p(z∣z′)

Q: How do we ensure that a given distribution is the stationary distribution of a Markov chain?

A: By constructing the transition probabilities to obey detailed balance:

p∗(z′)p(z∣z′)=p∗(z)p(z′∣z)(Sufficient condition - although not necessary)p^*(z')p(z|z') = p^*(z)p(z'|z) \hspace{3em} \text{\tiny (Sufficient condition - although not necessary)}p∗(z′)p(z∣z′)=p∗(z)p(z′∣z)(Sufficient condition - although not necessary)

We can convince ourselves that this is true by taking the sum over z′z'z′ on both sides:

∑z′p∗(z′)p(z∣z′)⏟lhs=∑z′p∗(z)p(z′∣z)⏟rhs=p∗(z)∑z′p(z′∣z)⏟normalized distributionsums to 1=p∗(z)\sum_{z'}\underbrace{p^*(z')p(z | z')}_{lhs} = \sum_{z'}\underbrace{p^*(z)p(z' | z)}_{rhs} = p^*(z)\underbrace{\sum_{z'}p(z' | z)}_{\begin{gather*}\text{\tiny normalized distribution} \\[-.5em] \text{\tiny sums to 1}\end{gather*}} = p^*(z)z′∑​lhsp∗(z′)p(z∣z′)​​=z′∑​rhsp∗(z)p(z′∣z)​​=p∗(z)normalized distributionsums to 1​z′∑​p(z′∣z)​​=p∗(z)

Markov Chains: ergodicity

In addition to being invariant, we also want to make sure that the Markov Chain actually converges to our target distribution as we let the length of the chain approach infinity.

For this to be true, the chain should be:
Irreducible. There should be a sequence of non-zero probability transitions from any state zsz_szs​ to any other state ztz_tzt​.
Aperiodic. The probability of returning to the same state should not have a periodic dependence on the number of steps.

If this is fulfilled, the chain is said to be ergodic.

Metropolis-Hastings

We need to come up with some p(z∣z′)p(z|z')p(z∣z′) that fulfills detailed balance:

Detailed balance equation: p(z′)p(z∣z′)=p(z)p(z′∣z)p(z')p(z|z') = p(z)p(z'|z) p(z′)p(z∣z′)=p(z)p(z′∣z)

Idea: factorize p(z∣z′)p(z|z')p(z∣z′) into: p(z∣z′)=q(z∣z′)⏟proposalα(z,z′)⏟acceptancep(z|z') = \underbrace{q(z|z')}_{\text{\tiny proposal}}\underbrace{\alpha(z, z')}_{\text{\tiny acceptance}}p(z∣z′)=proposalq(z∣z′)​​acceptanceα(z,z′)​​

We can now choose qqq freely, and then choose α\alphaα carefully to ensure detailed balance:

p(z′)q(z∣z′)α(z′,z)=p(z)q(z′∣z)α(z,z′)⇒α(z′,z)α(z,z′)=p(z)q(z′∣z)p(z′)q(z∣z′)p(z')q(z|z')\alpha(z', z) = p(z)q(z'|z)\alpha(z, z') \hspace{1em}\Rightarrow \hspace{1em}\frac{\alpha(z', z)}{\alpha(z, z')} = \frac{p(z)q(z'|z)}{p(z')q(z|z')}p(z′)q(z∣z′)α(z′,z)=p(z)q(z′∣z)α(z,z′)⇒α(z,z′)α(z′,z)​=p(z′)q(z∣z′)p(z)q(z′∣z)​

One way to satisfy this equation is:

α(z,z′)=min⁡(1,p(z)q(z′∣z)p(z′)q(z∣z′))\alpha(z, z') = \min\left( 1, \frac{p(z)q(z'|z)}{p(z')q(z|z')}\right)α(z,z′)=min(1,p(z′)q(z∣z′)p(z)q(z′∣z)​)

This is called the Metropolis-Hastings algorithm.

Common strategy: use Gaussian centered around current state.

Metropolis-Hastings - conclusion

Problem: We have a target density p(z)p(z)p(z) that we can evaluate, but not sample from.

We can do a random walk, according to a Markov Chain defined simply by a transition probability.
If we accept/reject appropriately, this will lead to samples from p(z)p(z)p(z).

Consider how amazing this is.

Note, however - that we do not get independent samples from p(z)p(z)p(z). This is the price we pay. Often, we thus subsample from our samples in order to get (roughly) independent samples.

Gibbs sampling (2)

In a Gibbs sampler, we repeatedly sample one of the variables conditioned on all the rest

Repeat:

Sample new value for ziz_izi​ from p(zi∣z∖i)p(z_i | z_{\setminus i})p(zi​∣z∖i​)
