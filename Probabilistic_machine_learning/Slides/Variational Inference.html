Approximate inference

Recap: What is "inference"?

Machine Learning terminology:

Learning typically refers to finding optimal parameters for variables in the model
Inference refers to the task of reasoning about unknown probability distributions.

In a fully Bayesian setting, one treats all parameters as distributions - so there is no learning, only inference.

For the same reason, in the Bayesian setting, there will often be no distinction between parameters θ\boldsymbol \thetaθ, and latent variables z\boldsymbol zz. Everything will be contained in z\boldsymbol zz.

What is "inference"? (2)

Central tasks:

Find the distributions over latent variables that best explain my data:
p(z∣x)⏟posterior=p(x∣z)⏞likelihoodp(z)⏞priorp(x)⏟evidence / marginal log-likelihood\underbrace{p(\boldsymbol z|\boldsymbol x)}_{\text{posterior}} = \frac{\overbrace{p(\boldsymbol x|\boldsymbol z)}^{\text{likelihood}}\overbrace{p(\boldsymbol z)}^{\text{prior}}}{\underbrace{p(\boldsymbol x)}_{\text{evidence / marginal log-likelihood}}}posteriorp(z∣x)​​=evidence / marginal log-likelihoodp(x)​​p(x∣z)​likelihood​p(z)​prior​​
Make predictions on new data by evaluating expectations over the posterior:
p(x′∣x)=∫zp(x′∣z)p(z∣x)dzp(\boldsymbol x'|\boldsymbol x) = \int_{\boldsymbol z} p(\boldsymbol x'|\boldsymbol z)p(\boldsymbol z|\boldsymbol x)d\boldsymbol zp(x′∣x)=∫z​p(x′∣z)p(z∣x)dz

Often, we will have no closed form for p(x)p(\boldsymbol x)p(x) and p(z∣x)p(\boldsymbol z|\boldsymbol x)p(z∣x), and both steps will be intractable.

We therefore need approximations. Markov chain Monte Carlo is one approach. But it is slow. What else can we do?

Laplace approximation

Laplace approximation

Goal: approximate (potentially unnormalized) density p~(z)\tilde p(z)p~​(z)

Idea: approximate by a Normal distribution at one of its modes

How do we do this?
Taylor expansion up to 2nd order at mode z0z_0z0​ ln⁡p~(z)≈ln⁡p~(z0)+(z−z0)∂ln⁡p(z)∂z∣z=z0⏟=0 since we are at a mode+12(z−z0)2∂2ln⁡p(z)∂z2∣z=z0⏟≔−1σ2\ln \tilde p(z) \approx \ln \tilde p(z_0) + \underbrace{(z-z_0)\frac{\partial \ln p(z)}{\partial z}\bigg\rvert_{z=z_0}}_{=0 \text{ since we are at a mode}} + \frac{1}{2}(z-z_0)^2\underbrace{\frac{\partial^2 \ln p(z)}{\partial z^2}\bigg\rvert_{z=z_0}}_{{\coloneqq}-\frac{1}{\sigma^2}}lnp~​(z)≈lnp~​(z0​)+=0 since we are at a mode(z−z0​)∂z∂lnp(z)​​z=z0​​​​+21​(z−z0​)2:=−σ21​∂z2∂2lnp(z)​​z=z0​​​​
so, p~(z)≈p~(z0)exp⁡(−12(z−z0)2σ2)∝1σ2πexp⁡(−12(z−z0)2σ2)=N(z0,σ2)\text{so, }\tilde p(z) \approx \tilde p(z_0)\exp\left(-\frac{1}{2}\frac{(z-z_0)^2}{\sigma^2}\right) \propto \frac{1}{\sigma\sqrt{2\pi}}\exp\left(-\frac{1}{2}\frac{(z-z_0)^2}{\sigma^2}\right) = \mathcal{N}(z_0, \sigma^2)so, p~​(z)≈p~​(z0​)exp(−21​σ2(z−z0​)2​)∝σ2π​1​exp(−21​σ2(z−z0​)2​)=N(z0​,σ2)

Note that σ>0\sigma > 0σ>0 implies that the second derivative must be negative (i.e. z0z_0z0​ must be a local maximum)

The Laplace approximation thus finds the fit by ensuring that the first and second derivates match at point z0z_0z0​.

Laplace approximation - example

Approximating a mixture model with two components (Student-t distributed) with Laplace approximations at the two modes.

Note that the approximation only summarizes the target distribution around one mode.

Also note how easy it is to evaluate the 2nd derivative (i.e. hessian) with autodiff!

import torch
import torch.distributions as D
from torch.autograd.functional import hessian

mix = D.Categorical(torch.tensor([0.6, 0.4]))
comp = D.StudentT(df=1,
                  loc=torch.tensor([1., 2.]),
                  scale=torch.tensor([.2, .5]))
mm = D.MixtureSameFamily(mix, comp)

x_mode = torch.tensor(1.)
sigma = (-hessian(mm.log_prob, x_mode))**(-1/2)
laplace_approx1 = D.Normal(x_mode, sigma)

Laplace approximation - conclusions

The Laplace approximation is a simple technique to approximate a probability density.

We have to pick a local maximum z0z_0z0​ at which to evaluate it. We would typically get this using some optimizer.

The main weakness of the method is that it uses information only from this single point z0z_0z0​, making it a very local procedure.

Variational inference

Variational inference: the general idea:

Approximate complex distribution with a simpler, tractable distribution

Measure similarity between target p(z)p(z)p(z) and approximation distributions q(z)q(z)q(z) using the Kullback Leibler divergence: KL(q||p)=∫q(z)ln⁡q(z)p(z)dz\text{KL(q||p}) = \int q(z) \ln\frac{q(z)}{p(z)}dzKL(q||p)=∫q(z)lnp(z)q(z)​dz

Red: Laplace approximation.
Green: Variational approximation.

We will see that this turns inference into an optimization problem: "inference", the reasoning about probability distributions of unknown variables, becomes a matter of finding the optimal probability distribution among a predefined family of distributions.

VI: Two scenarios

Partly variational (partly maximum likelihood):
We have some parameters which we characterize by distributions, others by point estimates. For instance, for mixture models, we modeled the latent variables using probability distributions, but had point estimates for the parameters in each component. We thus ultimately maximize p(x;θ)p(x; \theta)p(x;θ) wrt θ\thetaθ (like we did in EM).
Fully variational:
We describe all parameters using distributions. We find the optimal approximations qqq for all of them.

Minimizing KL(q(z)∣∣p(z∣x))\text{KL}(q(z)||p(z|x))KL(q(z)∣∣p(z∣x))

Summary: we need to minimize KL(q(z)∣∣p(z∣x))\text{KL}(q(z)||p(z|x))KL(q(z)∣∣p(z∣x))

For Gaussian mixture models, this was easy. We could simply choose q(z)=p(z∣x)q(z) = p(z|x)q(z)=p(z∣x)

For more complex models, p(z∣x)p(z|x)p(z∣x) will be intractable. So what can we do?

ln⁡p(x)= Ez∼q(z)(ln⁡p(x,z)q(z))⏟L(q)+ Ez∼q(z)(−ln⁡p(z∣x)q(z))⏟KL(q(z)∣∣p(z∣x))\begin{align*}\ln p(\boldsymbol x) =& ~\underbrace{\mathbb{E}_{\boldsymbol z \sim q(\boldsymbol z)} \left (\ln \frac{p(\boldsymbol x, \boldsymbol z)}{q(\boldsymbol z)} \right )}_{\mathcal{L(q)}} \\+&~ \underbrace{\mathbb{E}_{\boldsymbol z \sim q(\boldsymbol z)} \left(-\ln \frac{p(\boldsymbol z |\boldsymbol x)}{q(\boldsymbol z)} \right )}_{\text{KL}(q(z)||p(z|x))}\end{align*}lnp(x)=+​ L(q)Ez∼q(z)​(lnq(z)p(x,z)​)​​ KL(q(z)∣∣p(z∣x))Ez∼q(z)​(−lnq(z)p(z∣x)​)​​​

We can maximize L(q)\mathcal{L(q)}L(q) instead. Since the KL is non-negative, L(q)\mathcal{L(q)}L(q) will be a lower bound on p(x)p(x)p(x)

We call L(q)\mathcal{L(q)}L(q) the Evidence Lower Bound (ELBO)

Variational inference - Choosing q(z)q(z)q(z)

How do we choose a good q(z)q(z)q(z)?

Note that Eq(z)(ln⁡p(x,z)q(z))\mathbb{E}_{q(\boldsymbol z)} \left (\ln \frac{p(\boldsymbol x, \boldsymbol z)}{q(\boldsymbol z)} \right )Eq(z)​(lnq(z)p(x,z)​) is a functional in qqq: it takes a function q(z)q(z)q(z) as input and returns a value.

Finding the function that optimizes a functional can be achieve used "calculus of variations", which generalizes the notion of derivatives to functionals. I.e., how the value of a functional changes when you make small changes (variations) to the input function.

This is where the term "variational" comes from.

Choosing q(z)q(z)q(z): Free-form vs fixed-form

There are two distinct approaches to variational inference

Free-form:
No assumptions are made about the functional form of qqq (or the factors of qqq). These arise automatically as a consequence of the calculus of variations. This is the approach taken in Bishop. This is mathematically quite involved.
Fixed-form:
We limit ourselves to a specific family of distributions, and optimize the parameters of this family to find the optimal q(z)q(z)q(z). We can use standard optimization techniques for this. This is the approach used most often in practice.

We will consider only the second case here.

As you can see in Bishop, the free form approach often results in distributions from the exponential family, which is also the family typically chosen

Variational inference: General recipe

Choose parametric, tractable family of distributions
Among this family, find q(z)q(z)q(z) that minimizes the Kullback-Leibler divergence to the target p(z∣x)p(z|x)p(z∣x)

Mean Field family

We need to somehow make qqq managable.

Classic VI typically restricts itself to the Mean Field family - which assumes that q(Z)q(\boldsymbol Z)q(Z) factorizes completely

q(Z)=∏i=1N∏j=1Dqij(zij)q(\boldsymbol Z) = \prod_{i=1}^N\prod_{j=1}^D q_{ij}(z_{ij})q(Z)=i=1∏N​j=1∏D​qij​(zij​)

Note that Z={z1,z2,…,zn}\boldsymbol Z = \{\boldsymbol z_1, \boldsymbol z_2, \ldots, \boldsymbol z_n \}Z={z1​,z2​,…,zn​} corresponding to the latent variables associated with each data point X={x1,x2,…,xn}\boldsymbol X = \{\boldsymbol x_1, \boldsymbol x_2, \ldots, \boldsymbol x_n\} X={x1​,x2​,…,xn​}. We thus assume independence both between all zi\boldsymbol z_izi​ and zj\boldsymbol z_jzj​ and between the DDD different variables within each zi\boldsymbol z_izi​.

Each of these latent variables will have its own variational distribution.

Limitations of mean field

The disadvantage of mean field is that we do not capture correlations between variables.

In some cases, it will therefore be necessary to choose qqq distributions with more complex covariance structure.

Take-home messages:

  • Variational Inference (VI) attempts to approximate a posterior distribution (over latent variables and parameters) with a simpler distribution.
  • Mean-field VI assumes that this simpler distribution factorizes - so that the latent variables and parameters are independent.
  • In standard VI, we optimize these parameters for each individual data point. We'll see how this can be avoided in the next lecture.
